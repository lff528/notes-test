通用约定
属性值0的后面不要跟任何单位

尽量减少通配符(*)选择器的使用

不要在css文件中通过@import引入其他的css文件

css中不要使用!important:改控件样式的时候可以覆盖控件样式,自妍开发的代码不要出现

同一个文件或一个页面中引入的一组样式表文件中,禁止同样的规则出现多次

属性定义后必须以分号结尾

选择器的嵌套层级一般情况下不要超过3级

 

文件编码
统一使用无BOM的UTF-8编码

缩进
4个空格作为一个缩进层级

可以使用tab作为一个缩进层级,但前提是1个tab退格需要设置为4个空格

同一个层级的代码保持同一个缩进级别

.box {
    width: 200px;
    height: 200px;
}

空格
选择器和{之间要有一个空格

属性名和之后的冒号(:)不能有空格,冒号(:)和后面的属性值之间必须要留有一个空格

属性值为多个值时,即列表型时,属性值与属性值之间使用英文逗号(,)分隔,且逗号(,)和后面的属性之间要留有一个空格

font-family: Arial, sans-serif;
行尾不能有多余的空格

换行
每个属性单独占一行

{的前面不需要换行

{的后面和}的前面需要换行

注释和下面被注释的内容之间不留有空行

.box {
    color: #000;
    width: 200px;
    height: 200px;
    font-family: Arial, sans-serif;
}

注释
CSS中统一使用/*……*/的方式注释

禁止使用//的方式注释

注释与下面被注释的内容保持同样的缩进层级

模块注释,写在模块的上面

单行注释,写在被注释行的上面一行

注释和下面被注释的内容之间不留有空行

/* header */
.header {
    clear: both;
    width: 100%;
    overflow: hidden;
}
引号
css中需要使用引号的地方,最外层统一使用双引号

url统一使用引号

属性选择器中的属性值使用引号

/* 正确的使用引号 */
input[type="text"] {
    color: #f20;
    background: url("../images/bg-input.png") 0 0 no-repeat;
}
媒体查询
将媒体查询的规则考勤与它们相关的规则,不要把媒体查询放到页面的底部或者顶部,又或者是一个单独的文件

命名
选择器命名,统一使用小写字母

选择器名称由多个单词拼接时,单词之间使用横线分隔

.user-center {
    clear: both;
    width: 1000px;
    margin: 0 auto;
    overflow: hidden;
}

色值
色值取值统一为16进制小写字母,禁止使用rgb()

可以使用16进制简写形式的时候就使用简写形式

有透明度要求的地方使用rgba格式

/* 色值的正确使用方式 */
.box {
    color: #333;
    border: 1px solid #fdfcf8;
    background: rgba(0,0,0,0.8);
}
清除浮动
可以通过使用伪类或者触发BFC的方式进行清除浮动,不要通过在HTML添加空标签的方式清除浮动

!important
禁止使用!important属性

数值
当数值为小于1的实数时,不能省略整数部分的0

表达式
禁止在css中使用表达式

字重
font-weight属性统一使用单词normal和bold.

Icon
css字重分100-900共九档,但目前受浏览器和字体质量的限制,实际上仅仅支持400和700两档,即分别等价于关键词normal和bold.

选择器
在需要对元素进行样式控制的时候,优先使用类选择器或id选择器,尽量不使用元素选择器
